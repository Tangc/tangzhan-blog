** 1、从专家到Agent **
昨天有集中出现几篇推 Alpai 这个 Agent 的文章，比如 https://mp.weixin.qq.com/s/Wg02AD6BJKUzsIN3vKFMZg
和其他agent由资深开发者推出不同的是，作者闲人一坤是以制作AI视频闻名的，代表作是24年就推出的AI短剧《山海奇镜》。
我觉得里面可以看到两个关键点
***关键一、 如何用 Agent 解决复杂问题？***
- 工程化
它采用是按工程化思路，逐步拆解，像瀑布流一样，上一个流程的产出是下一个流程的输入，逐步完成。
- 推广一下
想一下，开发软件是否也是类似的问题，如果想用一个 Agent 产品解决复杂系统开发问题的话，这个拆解方式是否一种可行思路？

***关键二、 怎么寻找可做的 Agent 方向？***
- 从业务出发
大家的底座都是 LLM，但你对业务领域的积累、理解越深，方法越体系越具体越落地，以此为基础驱动 LLM 产出的结果，才能越深越有壁垒越有实际价值。
所以技术人员的破局法门之一就是多往业务走，理解业务甚至成为业务专家，最懂业务的技术专家，还怕自己没有不可替代性吗？
然后把业务知识通过 Agent 落地，就算公司不赏识，拿出来也是有市场的。


** 2、All you need is Speed **
看到关于 Hassan El Mghari 这个独立开发者的经验分享，对其中的观点深表认同，分享一下
他的github地址，https://github.com/nutlope，4年时间开发了40多款AI应用(1年10款我觉得是比较健康的高强度了)，积累了大几百万的用户，自然也有不菲的收入。
他分享的核心观点：
- LLM 给构建产品带来极大的便利。
- 独立开发构建产品就是要快，还是快。（雷军：7字真言(值得反复品味)，火云邪神：唯快不破）。
- 不要将产品复杂化，2周完成一款，上架很重要。随后是收集反馈和改进。
- 从任何生活细节中，时刻思考可以产品化解决的场景，按优先级挨个做就完事。

我读出来他没分享的：
- 快的基础是要有一套开箱即用的工具栈，要在构建产品的同时积累。
- 什么时候停止对产品的投入？我想是没有反馈，或者改进成本过高，或者底层LLM不够用的时候。
- 正确的时候做正确的事，当下就是要做AI产品，不要老抱着那破烂3件套不放。(雷军又来了：风口上🐷都给你吹起来)
